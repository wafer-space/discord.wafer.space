@everyone - People on the mailing list should have just received an email title "Late changing updates -- *Initial* GDS is still due today, final GDS TBD -- wafer.space GF180MCU Run #1" which explains the following:
> Earlier today we got feedback from GF that a number of DRC errors types we were planning to waive can not be waived. Leo (@Leo Moser (mole99)) is currently hard at work updating both the template and precheck to fix these issues.
> 
> The first set of DRC error fixes require changes to the seal ring and I/O pads generated by the template and Librelane. As these are part of the user design, everyone will need to pull the latest version and update their design. The precheck will be updated to verify these errors have been corrected in your submitted design.
> 
> All designs will also need to be 100% antenna DRC clean. The precheck is being updated to include these antenna checks. The template's LibreLane and OpenROAD tooling should already be fixing antenna issues and producing DRC clean results.
> 
> Lastly, the final DRC error fixes require changes to the fill generation. We hope that we are able to accommodate these changes and still meet density targets in all designs. As these are again part of the user design, everyone will need to pull the latest version of the fill generation and update their design. The precheck will again be updated to verify the fixed fill has been generated in your submitted design.
> 
> **I'm requesting everyone create their project on https://platform.wafer.space website and upload at least an *initial* GDS before the upcoming deadline.**
> 
> As we do not yet know the amount of impact from the updates nor the effect delays will have on our planned timeline with GF, I am not yet setting a new deadline for the final GDS but it is likely to be sometime next week.

[12/18/2025 14:19] mole99
@everyone 📢 The submission deadline has now passed!

Congratulations to everyone on the fantastic submissions.
If all goes well, the submission was the last active action required of you.

The next steps are as follows:

- We will stitch the reticle over the weekend. If we identify any violations that were not flagged by the precheck, we will contact you in time to resolve the issue.

- The final reticle will be stitched on Monday and sent to GF for validation. GF will then run their proprietary DRC deck on the reticle. If any violations are identified that were not caught by the open source DRC decks, we will contact you to resolve the issue.

We have rerun the latest precheck on all designs: some showed issues with the latest version 1.5.5. We have contacted the project authors, and they still have the opportunity to submit an updated design over the weekend.

If any of the other project authors catch a chip-breaking bug in their design, they will also have the chance to submit one final update - please only do so if it is important.

Now that the projects are submitted, please make sure to update:

- 📝 The project description.
- 🟩 The project visibility.
- 🌐 The project URL (if applicable).
- 🪪 And the project license.

This allows us to promote both wafer.space and all the great open source designs on this shuttle.
